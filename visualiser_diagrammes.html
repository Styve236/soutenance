2. Diagramme de Séquence (Processus de Commande)

sequenceDiagram actor Client participant Frontend as Interface (Navigateur) participant Backend as API PHP participant DB as Base de Données Client->>Frontend: Recherche un miel/boutique Frontend->>Backend: Requête API (search_api.php) Backend->>DB: SELECT sur restaurants (apiculteurs) et menus (miels) DB-->>Backend: Résultats (Données) Backend-->>Frontend: JSON résultats Frontend-->>Client: Affiche les suggestions en temps réel Client->>Frontend: Ajoute un miel au panier Frontend->>Backend: POST panier_api.php (action=add) Backend->>DB: INSERT / UPDATE table panier DB-->>Backend: Confirmation Backend-->>Frontend: Succès (Nouveau total) Frontend-->>Client: Met à jour le widget panier Client->>Frontend: Clique sur "Valider la commande" Frontend->>Backend: POST Checkout Backend->>DB: INSERT commande Backend->>DB: DELETE FROM panier (vider le panier de l'utilisateur) Backend->>DB: INSERT notification (ex: type='order') DB-->>Backend: Confirmation Backend-->>Frontend: Succès commande Frontend-->>Client: Redirection vers confirmation.php avec statut
